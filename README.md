# 人流アノマリー要因分析アプリ

日時と緯度・経度を入力すると、**OpenAI がWeb検索を横断して人流アノマリー（異常な増加）の原因を分析**し、Markdown形式のレポートを生成するモバイルWebアプリ（PWA）です。

位置情報アナリスト／データサイエンティストの視点で、
「どこで（Where）」「なぜ（Why）」を深掘りし、**「データで言える事実」と「Web情報からの推定」を区別**したレポートを作成します。

## 特長

- 📱 **PWA** — スマホのブラウザで開き、ホーム画面に追加すればネイティブアプリのように使えます（インストール不要）
- 🔎 **Web横断調査** — OpenAI の `web_search` ツール（Responses API）で、イベント・ニュース・災害/気象・交通・SNSなどをリアルタイムに調査
- 🧭 **位置の指定は2通り** — 「**緯度・経度**」または「**住所・地名**」をトグルで切り替えて入力（住所モードでは、Web検索で地点を特定して分析）
- 📍 **現在地取得** — ワンタップで端末のGPSから緯度経度を入力（緯度経度モード）
- 🔀 **2つの接続方法** — 「**プロキシ経由**（合言葉で利用・配布向け）」と「**OpenAI直接**（自分のキー・開発向け）」を設定で切替
- 🔐 **キーの扱い** — OpenAI直接モードではキーは端末内 `localStorage` にのみ保存。プロキシ経由モードではキーは**サーバー（プロキシ）側で保持**され、利用者は合言葉のみ入力
- ⚡ **ストリーミング表示** — 調査の進捗（検索クエリ・取得ページ）をリアルタイム表示し、レポートを逐次描画
- 📋 レポートの **コピー / Markdownダウンロード** に対応
- 🌗 ライト/ダークモード自動対応

## 使い方

1. アプリを開く（下記「公開方法」参照）
2. 上部「**API設定**」を開き、接続方法を選ぶ
   - **プロキシ経由（配布向け）**: 配布者から共有された**合言葉**を入力（プロキシURLは通常設定済み）。→ [プロキシの立て方は `proxy/README.md`](./proxy/README.md)
   - **OpenAI直接（開発向け）**: 自分の OpenAI APIキー（`sk-...`）を入力。キーは [platform.openai.com/api-keys](https://platform.openai.com/api-keys) で取得
3. 必要ならモデル名を、お使いのアカウントで利用可能な**Web検索対応モデル**に変更
4. **日時**を入力
5. 位置の指定方法を選択：
   - **緯度・経度**モード：緯度・経度を入力（「現在地を取得」も利用可）
   - **住所・地名**モード：住所・施設名・ランドマーク等を入力（地点はWeb検索で特定）
6. 任意で観測状況／周辺施設／周辺地形も入力
7. 「**レポートを生成**」をタップ
8. 調査完了後、レポートが表示されます（コピー／保存可能）

## 使用モデル

- OpenAI の **Responses API**（`POST /v1/responses`）を使用し、`tools: [{ "type": "web_search" }]` でWeb検索を有効化します。
- モデルは設定画面で指定・変更できます（既定 `gpt-5.6`）。**お使いのアカウントで利用可能なWeb検索対応モデル**に合わせてください（例: `gpt-5.x` 系）。
- ストリーミング（`stream: true`）で、検索の進捗とレポートを逐次表示します。プライバシー配慮のため `store: false` を指定しています。

> ⚠️ **CORS に関する注意**: OpenAI API にはブラウザ直叩きを明示許可するヘッダがありません。**OpenAI直接モード**では、公開URL（GitHub Pages等）からの呼び出しが CORS でブロックされる場合があります（`localhost` では概ね動作）。**プロキシ経由モード**を使えばこの問題は解消します。

## 他人に使ってもらう（方式B：プロキシ経由）

利用者に OpenAI キーを持たせず、**合言葉だけ**で使ってもらう構成です。

1. **Cloudflare Workers プロキシをデプロイ** — 手順は [`proxy/README.md`](./proxy/README.md)（キーとオリジン、任意でレート制限を設定）
2. `index.html` 冒頭の `const PROXY_URL = '';` に、デプロイした Worker の URL を設定してプッシュ
3. 利用者には **アプリのURL** と **合言葉** の2つを共有するだけ

OpenAIキーはプロキシのサーバー側（環境変数）に置かれ、アプリやブラウザには出ません。費用は配布者のOpenAIアカウントに発生するため、OpenAI側で**月次のUsage limit**を設定しておくことを強く推奨します。

## 公開方法（いずれか）

これは**ビルド不要の静的サイト**です。以下のいずれかで配信できます。

- **ローカル確認**: このディレクトリで `python3 -m http.server 8080` を実行し、スマホから `http://<PCのIP>:8080/` を開く
- **GitHub Pages**: リポジトリの Settings → Pages で本ブランチ／ルートを公開
- 任意の静的ホスティング（Netlify / Vercel / Cloudflare Pages / S3 など）にファイル一式をアップロード

> ⚠️ PWA（Service Worker・ホーム画面追加）は **HTTPS**（または `localhost`）でのみ有効です。GitHub Pages等はHTTPSで配信されるため問題ありません。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体（UI・API呼び出し・Markdownレンダラをすべて内包） |
| `manifest.webmanifest` | PWAマニフェスト |
| `sw.js` | Service Worker（アプリシェルのオフラインキャッシュ。API通信はキャッシュしません） |
| `icon.svg` / `icon-192.png` / `icon-512.png` | アプリアイコン |

外部ライブラリ・CDNへの依存はありません（Markdownレンダラも内蔵）。

## 注意事項

- レポートには **Web情報に基づく推定（仮説）** が含まれます。重要な判断の前に一次情報でご確認ください。
- APIの利用にはOpenAIの利用料金が発生します。
- 位置情報・入力内容・検索クエリは、レポート生成のためOpenAI APIに送信されます。
